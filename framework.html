Framework .NET Windows 4.7

Se trata de un instalador sin conexión. Los instaladores con y sin conexión son independientes. Si su intención es redistribuir alguno de dichos instaladores para la instalación de su propio producto o de su propia aplicación, se recomienda elegir el instalador web, ya que es más pequeño y normalmente la descarga es más rápida.

Microsoft recomienda usar el instalador web y no el paquete sin conexión siempre que sea posible por el rendimiento óptimo y los requisitos de ancho de banda. El paquete sin conexión se puede usar en los casos en que el instalador web no esté disponible por falta de conexión a Internet. El tamaño de dicho paquete es mayor que el del instalador web y, además, los paquetes de idioma no están incluidos



Las características WPF de .NET Framework 4.7 toman una dependencia de D3DCompiler_47.dll. El archivo D3DCompiler_47.dll no está presente de forma predeterminada en las plataformas de sistema operativo Windows 7 SP1, Windows 2008 R2 SP1 y Windows 2012. Por tanto, es necesario instalar D3DCompiler_47.dll en dichas plataformas para que las características WPF de .NET Framework 4.7 funcionen según lo esperado. Consulte el artículo sobre problemas conocidos para obtener detalles sobre la instalación de D3DCompiler_47.dll.

-En Windows 7 SP1/Windows Server 2008 R2 SP1, verá Microsoft .NET Framework 4.7 como producto instalado en Programas y características, en el Panel de control.
-En Windows Server 2012, lo encontrará como Actualización para Microsoft Windows (KB3186505) en Actualizaciones instaladas, en el Panel de control.
-En Windows 8.1/Windows Server 2012 R2, lo encontrará como Actualización para Microsoft Windows (KB3186539) en Actualizaciones instaladas, en el Panel de control.
-En Actualización de aniversario de Windows 10 y Windows Server 2016, lo encontrará como Actualización para Microsoft Windows (KB3186568) en Actualizaciones instaladas, en el Panel de control..
